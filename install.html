部署指南

本文档说明如何在 阿里云 Linux Ubuntu 22.04 服务器上部署 iKindle。

前置条件

  • 阿里云 ECS Ubuntu 22.04 LTS（公网 IP）
  • 域名 91api.vip 已注册并解析到 ECS IP（添加 A 记录 kindle）
  • ECS 安全组放行 80 / 443 端口

步骤 1：安装依赖

sudo apt update
sudo apt install -y nginx python3 python3-pip certbot python3-certbot-nginx
pip3 install cnlunar

步骤 2：克隆代码（手动）

由于 CLAUDE.md 全局规则禁止 git commit / git push，代码需手动复制到 ECS。

# 在 ECS 上：
mkdir -p /home/ikindle/{templates,deploy,assets,build}
# 复制本地文件到 ECS（rsync / scp）：
rsync -avz --exclude='.git' --exclude='build/' \
    ./ ikindle@<ECS_IP>:/home/ikindle/

步骤 3：配置 nginx

sudo cp /home/ikindle/deploy/nginx.conf /etc/nginx/sites-available/kindle
sudo ln -s /etc/nginx/sites-available/kindle /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

步骤 4：签发 HTTPS 证书

sudo certbot --nginx -d kindle.91api.vip

certbot 会自动修改 nginx 配置并 reload。证书 90 天有效，certbot systemd timer 自动续期。

步骤 5：配置 cron

sudo cp /home/ikindle/deploy/crontab /etc/cron.d/ikindle
sudo systemctl restart cron

cron 配置：每分钟跑 build.sh 生成最新 dashboard.html；每日 0:00 刷新 Hitokoto 缓存。

步骤 6：KWP4 实物验证

  1. 在 KWP4 浏览器打开 https://kindle.91api.vip/dashboard.html
  2. 截图确认布局（白底 / 纯黑加粗衬线 / 6 寸适配）
  3. 测试 meta refresh 300s（等待 5 分钟）
  4. 测试 XHR wttr.in（确认天气显示城市正确）
  5. 测试离线提示（关闭 WiFi 确认离线条显示）

故障排查

Q: 仪表盘打开但天气一直显示"加载中..."

A: 检查 /home/ikindle/build.log 看 curl wttr.in 是否成功；测试 curl https://wttr.in/ 是否可达。

Q: HTTPS 证书过期

A: sudo certbot renew 手动续期；检查 systemd timer systemctl list-timers | grep certbot。

Q: KINDLE 浏览器加载不到样式

A: 检查 nginx 配置文件 include /etc/nginx/mime.types; 是否存在；CSS MIME 类型应为 text/css。

本地预览（无 KINDLE 设备）

cd /home/ikindle
python3 -m http.server 8000
# 浏览器打开 http://localhost:8000/dashboard.html
